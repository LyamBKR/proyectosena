Diagrama de Clases del Proyecto de Software

En nuestro proyecto elaboraremos un diagrama de clases aplicando buenas prácticas de diseño orientado a objetos, es importante seguir unos pasos claves y considerar ciertos elementos estructuras y diseños.

📐 25.1 Diagrama de Clases para Nuestro Proyecto

El diagrama de clases define la estructura del sistema, mostrando clases con sus atributos y métodos, así como la relación entre ellas tales como herencias, asociaciones y otros.

En nuestro videojuego las clases que incluiremos son:

🎮

1. Jugador (Player)

Representa al usuario o personaje principal con atributos como salud, posición, puntaje y métodos para moverse o atacar.

👾

2. Enemigos (Enemy)

Diferentes tipos de enemigos. Con atributos y acciones propias.

✨

3. PowerUps o ítems

Las clases que representan objetos que el jugador puede recoger.

📈

4. Nivel (Level)

Controlamos el escenario, enemigos y condiciones de finalización.

🎯

5. Controlador del juego (GameController)

Maneja la lógica general y el estado del juego.

⭐ 25.2 Buenas Prácticas de Diseño Orientado a Objetos Aplicados

🔒

1. Encapsulamiento

Atributos privados o protegidos con método público de acceso.

🔄

2. Herencias

Uso de herencias para reutilización, por ejemplo, la clase Enemy como base para tipos específicos de enemigo.

🧩

3. Polimorfismo

Manejar diferentes comportamientos a través de interfaces o métodos sobrecargados.

🏭

4. Patrones de diseño

Aplicación de patrones de diseño como Factory para crear enemigos o ítems, y Observer para eventos del juego.

📝

5. Nombres claros

Nombres claros y constantes para clases, atributos o métodos.

⚡

6. Separación lógica

Separación lógica de responsabilidad según el principio de responsabilidad única.

Estas prácticas aseguran que el diagrama refleje un diseño sólido, mantenible y escalable para nuestro videojuego.

📊 25.3 Diagrama

A continuación, revisaremos el diagrama visual:

<<Interface>>
Personaje
+ actualizar()
Personaje
- salud: int
- posicion_x: List<PowerUp>
Enemigo
- Salud: int
+ Recibir_Powerups()
implements
uses

Leyenda del Diagrama:

Línea punteada:

Implementación (implements) - La clase Personaje implementa la interfaz Personaje

Línea sólida:

Asociación (uses) - La clase Personaje utiliza la clase Enemigo

Interfaz:

Representada con <<Interface>> y borde azul

Clase:

Representada con borde verde (Personaje) o rojo (Enemigo)

Descripción del Diagrama:

  • Interfaz Personaje: Define el contrato con el método actualizar() que debe ser implementado.
  • Clase Personaje: Implementa la interfaz Personaje y contiene los atributos salud: int y posicion_x: List<PowerUp>.
  • Clase Enemigo: Representa a los enemigos del juego con Salud: int y el método Recibir_Powerups().
  • Relaciones: La clase Personaje implementa la interfaz Personaje y mantiene una asociación con la clase Enemigo.
