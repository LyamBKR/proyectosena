📝 Ingeniería de Requisitos

Proceso de Ingeniería de Requisitos

La ingeniería de requisitos es una disciplina fundamental en el desarrollo de software que se encarga de identificar, analizar, documentar y gestionar las necesidades del cliente y los usuarios del sistema.

¿Qué es la ingeniería de Requisitos?

🎯

Definición Principal

"El proceso de recopilar, analizar y verificar las necesidades del cliente para un sistema de software es llamado Ingeniería de Requerimientos. La meta de la ingeniería de requerimientos es entregar una especificación de requerimientos de software correcta y completa."

🔍

Propósito

"La ingeniería de requerimientos apunta a mejorar la forma en que comprendemos y definimos sistemas de software complejos. Existen varias definiciones de requerimientos"

📋

Definición Técnica

"La ingeniería de requisitos se define como el proceso de definición, documentación y mantenimiento de los requisitos. La disciplina incluye todas las técnicas, métodos y procedimientos relacionados con la definición y gestión de las necesidades de los usuarios relacionadas con el sistema en estudio"

🔑 Actividades Claves para Obtener Requisitos

📊

Clasificación y Priorización

Se realizará unas clasificaciones de los requisitos de una categoría y se prioriza el requisito de los procesos e identificar cosas las cuales no sean importantes para el proceso

Clasificar requisitos por funcionalidad
Priorizar según importancia del negocio
Identificar requisitos no críticos
Establecer dependencias entre requisitos
📝

Documentación

Aquí se documenta toda la información para realizar el proceso. Revisar criterios importantes y evitar ambigüedades

Documentar requisitos de forma clara
Eliminar ambigüedades en la redacción
Establecer criterios de aceptación
Crear especificaciones detalladas
👥

Análisis de Stakeholders

Se revisan los Stakeholders y se revisan las necesidades del usuario en ello se revisa restricciones importantes como el tiempo de entrega o el presupuesto

Identificar todos los stakeholders
Analizar necesidades de usuarios finales
Evaluar restricciones de tiempo
Considerar limitaciones de presupuesto
🔄

Control de Cambios

Se realizan control de cambios en el momento de realizar el proceso con la trazabilidad de requisitos adicional se revisará el proceso y los procesos anteriores

Implementar sistema de control de versiones
Mantener trazabilidad de cambios
Revisar impacto de modificaciones
Documentar decisiones de cambio
🏢

Requisitos de Negocio

Aquí se revisa los requisitos del negocio o necesidades estrategias de la organización

Analizar objetivos estratégicos
Identificar necesidades organizacionales
Evaluar impacto en procesos de negocio
Considerar métricas de éxito

🔄 Proceso de Ingeniería de Requisitos

🔍

Análisis

Identificación y análisis de necesidades

En esta fase se identifican y analizan las necesidades del cliente y usuarios del sistema.

📝

Especificación

Documentación detallada de requisitos

Se documentan de manera detallada todos los requisitos identificados.

✅

Validación

Verificación y aprobación de requisitos

Se verifica y aprueba que los requisitos cumplan con las necesidades.

🔄

Gestión

Control de cambios y trazabilidad

Se gestionan los cambios y se mantiene la trazabilidad de los requisitos.
