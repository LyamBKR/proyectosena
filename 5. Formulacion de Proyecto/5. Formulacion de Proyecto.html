📋 Formulación de Proyecto

La formulación de proyecto es una etapa fundamental que define los objetivos, alcance y metodología para el desarrollo de nuestro videojuego. Esta fase establece las bases para todo el proceso de desarrollo.

🎯
Objetivos
📊
Alcance
🛠️
Metodología
🎮
Videojuego

🎯 5.1 Objetivo General

🎮Desarrollo de Software de Videojuegos

Crear y desarrollar un software de Videojuegos con la calidad que exige la industria actualmente que sea sencillo y fácil para el usuario, también integrar modelos dinámicos para personas en condición de discapacidad conforme a las opciones de accesibilidad de las consolas y plataformas.

🎯 5.2 Objetivos Específicos

1

Análisis y Desarrollo

Análisis y desarrollo de un nuevo Software de Videojuegos

2

Demostración de Habilidades

Crear un producto que muestre habilidades técnicas y creativas del equipo de desarrollo

3

Desarrollo Efectivo

Desarrollar las actividades para el desarrollo efectivo del software

4

Cobertura Integral

Abarcar todo el proceso de desarrollo, tanto económico como del software

📏 5.3 Alcance

Objetivo de Desarrollo

Construir un videojuego con funcionalidades básicas con sus funcionalidades básicas e interfaz antes del 17 de febrero de 2027.

📝 5.4 Captura de Requisitos

Para la captura de requisitos en el desarrollo de nuestro videojuego, nos enfocaremos en las historias de los usuarios y visualizar el flujo de un storyboard los cuales utilizaremos herramientas como:

📊

Jira

Gestión de historias de usuario

📋

Trello

Organización de tareas

🎨

Figma/Miro

Elaboración de Storyboards

📄

Google Docs

Documentación colaborativa

Decisión Final: Después de revisar utilizaremos la opción de Jira para la gestión de historias de usuario y Miro para la elaboración de un Storyboard

Definición de Captura de Requisitos

"La captura de requisitos se refiere de dónde vienen los requisitos y como se realizará la obtención de estos, en este proceso el desarrollador por primera vez podrá tener una comprensión adecuada del problema que se requiere solucionar, en esta etapa es importante la participación de todos los agentes mencionados en la lección anterior. Es la etapa más temprana en el ciclo de vida Proceso de vida que hace referencia a las etapas por las cuales pasa un desarrollo de software realizando una analogía del ciclo de la vida de una persona. del desarrollo del aplicativo web"

⚙️ 5.5 Definición de Funcionalidades

Definimos funcionalidad utilizando una plantilla clásica de historias de usuarios utilizando las plantillas básicas de estas historias.

Ejemplos de Historias de Usuario:

🎮

Como Jugador quiero controlar mi progreso para poder continuar con tranquilidad cada nivel

💾

Como usuario quiero guardar mi progreso para después

🖥️

Como jugador quiero poder tener un HUD muy intuitivo y fácil de entender

Estas historias se pueden registrar en Jira como tarjetas y agrupar por módulos del videojuego.

📋 5.6 Especificaciones de Captura de Requisitos

La técnica aplicada será captura de mediante historias de usuarios, combinando sesiones de Coworking y elaboración de Storyboards en Miro el cual incluye:

1

Reunión con Stakeholders

Identificación de necesidades

2

Redacción Colaborativa

Redacción colaborativa con las historias de usuarios

3

Traducción Visual

Traducción visual de historias de usuario clave en Storyboards que muestren escenas importantes junto con flujos de juegos importantes

📄 5.7 Plantillas

En esta plantilla revisaremos algunas historias de usuario:

Plantilla de Historia de Usuario

Como _____ (Tipo de usuarios) _______ Quiero ___(Funcionalidades)__ para _____(Objetivo)______

Criterios de aceptación:

  • Criterio 1
  • Criterio 2

Notas Adicionales

{Observaciones, Ideas y demás}

Storyboard:

  • Escena: Describe lo que sucede
  • Actores: Quienes Participan
  • Interacción: Que sucede o que reacción nos da
  • Resultado (Esperado): Experimentación del usuario

Ejemplo de Storyboard

Escena Actores Interacción Resultado
Pantalla de Presentación Jugador/Usuario Seleccionar una nueva partida Inicia una nueva partida
Selección de personaje Jugador/Usuario Avatar para el juego Personaje listo para Jugar
Tutorial Jugador/QA Superar los desafíos básicos Aprender los controles del Juego

Para la formulación de este proyecto se fundamenta en la captura estructurada de requisitos usando historias de usuarios y Storyboards, asegurando claridad y movimientos entre equipos y los jugadores

⚙️ 5.8 Funcionalidades

Funcionalidades del Videojuego

Sumando cada historia, se pueden adjuntar modelos de Storyboards visualizando la experiencia en Miro, donde las funcionalidades del juego deben ser claramente listadas como resultado de la captura de requisitos combinando las historias de usuarios y paneles visuales
